# ⚡ Cyberpunk Systems Engineering Portfolio

A high-performance, responsive developer portfolio featuring a futuristic cyberpunk aesthetic. Built entirely using vanilla technologies to showcase low-level frontend optimization, interactive canvas render loops, and client-side internationalization (i18n).

## 🚀 Key Architectural Features

- **Matrix-Inspired Digital Rain Engine:** A high-performance animation engine written natively in JavaScript utilizing the HTML5 Canvas API. It features real-time window boundary resizing, custom character mapping (alphanumerics & Japanese Katakana), alpha-blended trailing effects, and concurrent drop-interval scheduling.
- **Client-Side Localization Node (i18n):** A clean vanilla JavaScript internationalization engine that intercepts DOM properties to swap full context layouts dynamically between English (EN) and Spanish (ES) without requiring a server reload or framework dependencies.
- **Advanced Cyberpunk Visual Stack:** Handcrafted UI components utilizing pure CSS variables for modular theme control, neon glowing vectors (`neon-cyan`, `neon-magenta`), custom hardware-scanned scanline layers, dynamic grid backdrops, and hardware-accelerated text glitch modifiers.
- **Fluid Layout & Typography Responsive Array:** Utilizes advanced CSS layout techniques including flexible grids, flexible viewports, and modern scaling formulas (`clamp()`) to ensure complete visual fidelity across extreme device form factors (mobile layouts up to wide high-resolution desktop terminals).

## 🛠️ Technology Stack

- **Core Structure:** HTML5 (Semantic and fully accessible structure)
- **Style Architecture:** CSS3 (Custom properties, grid systems, advanced hover scaling states, and micro-animations)
- **Behavior & Render Pipeline:** Vanilla JavaScript (ES6+ Runtime, requestAnimationFrame loops, and interactive DOM manipulation APIs)
- **Typography:** Orbitron, Rajdhani, and Fira Code integration via Google Fonts.

## 💻 Development & Deployment Setup

### Quick View / Standalone Mode
Because this portfolio relies entirely on native browser features, it has zero external dependencies or compilation pipelines. 
1. Clone this repository locally.
2. Open the `index.html` file directly in any modern high-performance web browser.

### Integrated Development Deployment
To evaluate or build upon the localization schemas or canvas states locally, you can serve the directory using a lightweight HTTP server utility:

```bash
# Using Python
python -m http.server 8000

# Using Node.js (http-server)
npx http-server -p 8000
```
Once initialized, establish a loopback connection on your machine via `http://localhost:8000`.
